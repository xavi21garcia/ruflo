## Phases

### Phase R-1 — Shared security primitives (small, low risk)

- [x] **R-1.1** Add `@claude-flow/security` as workspace dependency in `v3/goal_ui/package.json` (`"file:../@claude-flow/security"`). Run `npm install`. DoD: `import { SafeStringSchema } from '@claude-flow/security'` resolves in tsx.
- [x] **R-1.2** Refactor `functions/_lib/sanitize.ts`: `wrapUserInput` becomes a thin wrapper that uses `@claude-flow/security`'s `SafeStringSchema` to assert non-control-character cleanliness BEFORE wrapping. Keep the `<user_input>` delimiter behavior. DoD: existing 8/8 `check:handler-fallback` still passes; new test covers control-char rejection.
- [x] **R-1.3** Replace handler-side ad-hoc string validation with shared schemas where applicable: `category` in generate-research-goal (use `IdentifierSchema`-style), `goal` in research-step (`SafeStringSchema`). DoD: all 4 handlers pass `check:handler-fallback`; live curl smoke returns 200 for happy + 400 for malformed.

### Phase R-2 — AgentDB memory bridge (medium)

- [x] **R-2.1** Inventory `@claude-flow/memory` browser-compatible exports. DoD: `docs/integration-r2-survey.md` documents which adapter (sql.js? IndexedDB?) goal_ui browser side can use.
- [x] **R-2.2** Add `@claude-flow/memory` workspace dep + create `src/integrations/agentdb/client.ts` thin browser-side wrapper (lazy-load). DoD: smoke test from page.evaluate writes+reads an entry.
- [x] **R-2.3** Migrate ONE persisted slot (`widgetConfig`) from RVF-only to AgentDB-backed via the new wrapper. Keep RVF as the on-disk format. DoD: `getWidgetConfig` returns the same shape; reload preserves; perf p95 ≤ 5ms.
- [x] **R-2.4** Add `searchPastGoals(query)` to `goalRepo` using HNSW recall over saved goals. DoD: typing into the goal field shows top-3 prior goal suggestions; `<10ms p95`.

### Phase R-3 — Swarm-driven research step (medium, big quality gain)

- [x] **R-3.1** Add `functions/_lib/swarm.ts` thin adapter: takes `{goal, stepTitle}` and returns a 4-agent specialized swarm result via `npx @claude-flow/cli` CLI invocation (researcher + analyst + critic + scribe). DoD: smoke test fires real CLI in mock mode → returns merged structured findings.
- [x] **R-3.2** Wire `functions/research-step/handler.ts` to use `swarm.ts` when env `RUFLO_USE_SWARM=true`; otherwise fall back to single-call path. DoD: env-flag e2e test toggles both paths.
- [x] **R-3.3** A/B harness: `scripts/check-swarm-quality.mjs` runs both paths against 3 seed goals; reports unique-citation-count + Anthropic-judged hallucination rate. DoD: report in `docs/swarm-ab-results.md`.

### Phase R-4 — Intelligence trajectory recording

- [x] **R-4.1** Trajectory schema: `{goalHash, configHash, perStepFindings[], finalReport, userVerdict}`. Persist on every research run completion. DoD: 5 trajectories stored, schema-validated.
- [x] **R-4.2** Wire `mcp__claude-flow__hooks_intelligence_trajectory-{start,step,end}` calls at the natural boundaries in `Index.tsx::executeResearch()`. DoD: trajectory event log visible via `npx @claude-flow/cli hooks intelligence stats`.
- [x] **R-4.3** Use trajectory recall in `GoalInput.tsx`: on input change, query intelligence-pattern-search for top-3 similar past goals; surface as autocomplete chips. DoD: chip-click prefills the field.

### Phase R-5 — Expose goal_ui as MCP server

- [x] **R-5.1** Define MCP manifest `mcp-server.json` listing 4 tools: `generate_research_goal`, `research_step`, `generate_action_items`, `optimize_research_config`, plus aggregate `run_full_research`. DoD: schema validates against MCP spec.
- [x] **R-5.2** Implement `functions/mcp/server.ts` (stdio MCP server) that proxies tool calls to the existing handlers. DoD: `claude mcp call goal_ui-research run_full_research --goal "..."` returns structured output.
- [x] **R-5.3** Add `npm run mcp:start` + integration test using `@modelcontextprotocol/sdk`. DoD: round-trip test passes.

### Phase R-6 — Hive-mind consensus on contested findings

- [x] **R-6.1** When R-3 swarm produces findings whose confidence-deltas exceed 0.2, kick off a 5-node `npx @claude-flow/cli hive-mind` Byzantine quorum vote. DoD: dissent-rationale recorded with each kept/dropped finding.
- [x] **R-6.2** `scripts/check-consensus.mjs` exercises 5-node swarm with 1 forced-faulty node; verifies vote terminates correctly. DoD: 5 simulated runs all reach consensus within 3 rounds.

### Phase R-7 — Hooks-driven CI

- [x] **R-7.1** GitHub Actions workflow `.github/workflows/goal_ui-hooks.yml` running on PR: `audit` worker + `testgaps` worker. DoD: workflow passes on this branch.
- [x] **R-7.2** Nightly cron job: `document` worker regenerates `docs/ui-inventory.md`; auto-PRs the diff. DoD: first nightly run produces a PR.
- [ ] **R-7.3** Build-size watcher: `optimize` worker fires when bundle grows >5%. DoD: simulated 5%-growth commit triggers worker.

## Out of scope

- Full claude-flow daemon embed in browser
